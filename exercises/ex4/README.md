# Create Interface Request  

In this part if the exercise, you are taking the persona of a Integration Developer. As a Integration Developer, you determine the right integration technology by processing an interface request. Integration Developer also gets a proposal of the integration technology with the best fit to the specified requirements and decides on the integration technology.

## Steps for creating Interface Request

Run through the following steps from the Integration Assesement Tool Landing Page:

1. Click **Requests** to go back to main landing page
<br><img src="/exercises/ex4/images/1.0_Ex3_Create_Request.PNG" width=80% height=80%>

2. Click **Interface Request**


3. Find Interface Requested created by you in previous exercise. This would be **Order2Cash_XX** where XX stands for your group number. Click **Requirement Analysis**


4. Step 1 is Integration Use Case Patterns. Click **A2A Integration**. Click **Step 2**


5. Step 2 is Transformation. Click **Yes**


6. Click: **Simple Transformation**


7. Click: **No**. Click: **Step 3**


8. Step 3 is Integration Content. Click: **Create**


9. Select on your scenario name which in the case of instructor is names **Order2Cash-1**


10. The system will connect to API Business Hub to get the content


11. Content from API Business Hub is displayed.


12. Enter **Customer** in the search field. Select **Target Application**


13. Select **iFlow Account Overview**. Scroll down


14. Select other iFlow **Alerts Get**. Select **Import**


15. Click **Step 4**. Step 4 is Connectivity


16. This question is about protocol conversion from source system to target system. For our interface request, this would be Yes. Click **Yes**


17. This question is for managing Trading Partner Management TPM. Since our Integrface Request does not have the need for TPM, we slelect **No** here


18. This question is asking for type of connectors required. Our scenario would need both, so selecting both.
- Select **Application Connectors**
- Select **Technology Connectors**
- Click **Step 5** (Step 5 is Monitoring and Operations)


19. This question is to provide monitoring ability to Business Users or no. Select **Yes**


20. This question is to provide monitoring ability forward technical errors or no. Select **Yes**


21. Click **Review** This gives youâ€™re the ability see the responses entered and Edit if needed. 


22. Click **Go to Technology Selection**


23. Click **Select Technology**. Here you can see the requts of your requirement analysis


24. Selected Technologies are higlighted. Select relevant instance from the dropdown:
- For SAP Application Interface Framework select **AIF**
- SAP Integration Suite, Cloud Integration, select **CI**
**Note** These are the instance configured in previous steps.
- Select **Submit**


25. Check if your Technology Name shows Completed. Click **Requests** to go back to the main landing page
